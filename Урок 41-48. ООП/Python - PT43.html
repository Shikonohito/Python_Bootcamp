Практическое Задание 43.1

Создайте класс "Task". Этот класс должен хранить поля:

  • Название
  • Описание
  • Статус ("выполнен"/"не выполнен")
  • Приоритет (1-9)

Для класса определите методы:

  • Конструктор
  • Строковое представление
  • Свойства
  • Повышение приоритета
    • Ничего не принимает.
    • Если приоритет меньше 9, то увеличивает приоритет на 1.
  • Понижение приоритета
    • Ничего не принимает.
    • Если приоритет больше 0, то уменьшает приоритет на 1.
  • Смена статуса
    • Ничего не принимает.
    • Если статус "выполнен", то меняет на "не выполнен".
    • Если статус "не выполнен", то меняет на "выполнен".

Создайте класс "TaskManager". Этот класс должен хранить поля:

  • Список задач

Для класса определите методы:

  • Конструктор
  • Геттер
  • Поиск задачи
    • Принимает название задачи.
    • Если задача с указанным названием найдена, то возвращает индекс задачи в списке.
    • В противном случае возвращает -1.
  • Добавление задачи
    • Принимает экземпляр класса "Task".
    • Если задача не найдена в списке, то добавляет её в список задач. Возвращает True.
    • В противном случае возвращает False.
  • Удаление задачи
    • Принимает название задачи.
    • Если задача найдена в списке, то удаляет её из списка задач. Возвращает True.
    • В противном случае возвращает False.
  • Фильтр по выполненным или не выполненным и приоритету
    • Принимает необязательный флаг (по умолчанию 0) и приоритет (по умолчанию 0).
    • Если флаг 0, то составляет список задач, которые выполнены или не выполнены.
    • Если флаг 1, то составляет список задач, которые выполнены.
    • Если флаг 2, то составляет список задач, которые не выполнены.
    • Если приоритет 0, то составляет список задач с любым приоритетом.
    • Если приоритет любое другое число, то составляет список задач с указанным приоритетом.
    • Возвращает полученный список задач в виде кортежа.

Практическое Задание 43.2

Используя классы из Практического Задания 43.1, реализуйте следующее меню: 

1. Показать все задачи
2. Показать задачу по названию
3. Показать задачи по критериям
4. Добавить задачу
5. Удалить задачу
6. Завершить программу

Все задачи хранятся в экземпляре класса "TaskManager". Задача – это экземпляр класса "Task".

Выбран пункт 1:

  • Программа проходит по каждой задаче и выводит в консоль её строковое представление в виде: 
    {название} {статус} {приоритет}
{название} {статус} {приоритет}
...

Выбран пункт 2:

  • Программа запрашивает с консоли название.
  • Если задача с указанным названием найдена в списке задач, то выводит в консоль её строковое представление и описание в виде: 
    {название} {статус} {приоритет}
{описание}
  • В зависимости от результата, в консоли выводится соответствующее сообщение.

Выбран пункт 3:

  • Программа запрашивает с консоли статус (0 – любой, 1 – выполнена, 2 – не выполнена) и приоритет (0 – любой, 1-9 приоритеты).
  • Если задачи по указанному фильтру найдены, то выводит в консоль строковое представление задач в виде: 
    {название} {статус} {приоритет}
{название} {статус} {приоритет}
...

Выбран пункт 4:

  • Программа запрашивает с консоли название, описание и приоритет.
  • Формирует из данных задачу.
  • Пытается добавить задачу в список задач.
  • В зависимости от результата, в консоли выводится соответствующее сообщение.

Выбран пункт 5:

  • Программа запрашивает с консоли название.
  • Пытается удалить задачу из списка задач.
  • В зависимости от результата, в консоли выводится соответствующее сообщение.
