Задание 1.
Создайте класс "Книга". Укажите в ней поля "Идентификатор", "Название", "Автор", "Страницы", "Количество на складе", "Цена за штуку".
Определите для класса конструктор и строковое представление, а также свойства.
Определите для класса __eq__ (оператор ==) или метод сравнения на равно. Он должен вернуть результат сравнения "Идентификатор" у разных книг.


Задание 2. (на основе задания 1)
Создайте класс "База данных". Укажите в ней поле "Список книг".
Определите для класса конструктор, а также свойства.
Реализуйте методы добавления, поиска, изменения и удаления книг.

Метод добавления книг должен сперва проверять наличие этой книги в списке.
Если книга есть, то у книги в списке должно увеличиваться "Количество на складе" на то количество, которое указано в книге при добавлении.

Учесть разные проверки для классов.


Задание 3. (на основе заданий 1 и 2)
Реализуйте следующее меню:
1. Показать список книг
2. Добавить книгу
3. Изменить книгу по идентификатору
4. Удалить книгу по идентификатору
5. Купить книгу
6. Завершить программу

Программа завершается только в том случае, если выбран пункт 6.

При выборе пункта 2 программа запрашивает ввод из консоли "Идентификатор".
Если такой идентификатор уже есть в базе данных, то
программа запрашивает только количество. После этого формирует объект "Книга" по данным из базы данных, используя идентификатор,
а потом добавляет книгу в базу данных.
Если же идентификатора нет, то программа запрашивает ввод из консоли "Название", "Автор", "Страницы", "Количество", "Цена за штуку",
а потом добавляет эту книгу в базу данных.

При выборе пункта 3 программа запрашивает ввод из консоли "Идентификатора".
Если такой идентификатор есть, то программа сперва выводит в консоль информацию о книге, а потом запрашивает ввод полей из консоли.
Если в поля вводить пустые данные, то они не должны меняться.

При выборе пункта 4 программа запрашивает ввод из консоли "Идентификатора".
Если такой идентификатор есть, то программа удаляет книгу из базы данных.

При выборе пункта 5 программа запрашивает ввод из консоли "Идентификатора".
Если такой идентификатор есть, то программа запрашивает ввод из консоли, сколько книг нужно отнять.
После этого программа отнимает из базы данных у этой книги количество и выводит итоговую цену в консоль.

Реализовать разные проверки на ввод данных.
