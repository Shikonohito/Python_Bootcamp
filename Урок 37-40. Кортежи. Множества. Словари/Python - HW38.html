Домашнее Задание 38.1

Напишите функцию find_occurrences, которая принимает кортеж строк и последовательность символов. Затем функция добавляет в список строку, в которую входит последовательность символов. В конце функция превращает список в кортеж и возвращает его.

Например:
Кортеж: ("banana", "apple", "bananamango", "mango", "strawberry-banana")
Последовательность символов: "ban"
Результат работы функции: ("banana", "bananamango", "strawberry-banana")

Подсказка:

Код и алгоритм поиска последовательности символов можете посмотреть на 24-м уроке.

Домашнее Задание 38.2

В магазине "Всё Не Дорого" запустили акцию по розыгрышу дорогой электроники. Для этого пользователи должны были вписать свою почту на сайте и нажать кнопку "Участвовать". В итоге собрался список почт, однако, если пользователь несколько раз вписывал одну и ту же почту и снова нажимал кнопку "Участвовать", то данные снова добавлялись в список. Более того, программа принимала любые строки, поэтому в списке есть мусорные данные. Напишите функцию check_mails, которая принимает список строк, удаляет дубликаты, проверяет правильность почты и возвращает кортеж из строк, которые прошли проверку. Общий верный формат почты "локальная-часть@домен".

Локальная часть может содержать:

  • латинские буквы (A–Z, a–z)
  • цифры (0–9)
  • спецсимволы: ! # $ % & ' * + - / = ? ^ _ { | } ~ `
  • точку (.), но не в начале, не в конце и не подряд

Доменная часть может содержать:

  • латинские буквы (A–Z, a–z)
  • цифры (0–9)
  • дефис (-), но не в начале и не в конце
  • домен верхнего уровня (например, .com, .ru, .org, .edu.az) в конце

Домашнее Задание 38.3

Напишите программу, которая запрашивает с консоли имена пользователей первого приложения через пробел, затем на следующей строке запрашивает с консоли имена пользователей второго приложения через пробел. Программа должна вывести в консоль следующие данные:

  • Имена пользователей обоих приложений
  • Имена пользователей только первого приложения
  • Имена пользователей только второго приложения
  • Имена всех пользователей

Пример работы в консоли:

Введите имена пользователей первого приложения через пробел: Teston Bob Tom Jack
Введите имена пользователей второго приложения через пробел: Bob Kate Jack Astrid

Пользователи обоих приложений: Bob Jack
Пользователи только первого приложения: Teston Tom
Пользователи только второго приложения: Kate Astrid
Все пользователи: Teston Bob Tom Jack Kate Astrid

Домашнее Задание 38.4

Реализуйте следующее меню: 

1. Показать данные всех баскетболистов
2. Показать только имена всех баскетболистов
3. Найти рост по имени баскетболиста
4. Добавить нового баскетболиста
5. Удалить баскетболиста по имени
6. Изменить рост по имени
7. Завершить программу

Данные баскетболистов хранятся в виде одного словаря, в котором ключи являются именами, а значения – ростом: 

{"Jack": 153, "Tom": 223, ...}

Выбран пункт 1:

  • Программа выводит в консоль данные баскетболистов в следующем формате: 
    {имя} - {рост}
{имя} - {рост}
...

Выбран пункт 2:

  • Программа выводит в консоль имена баскетболистов в следующем формате: 
    {имя}, {имя}, ...

Выбран пункт 3:

  • Программа запрашивает с консоли имя.
  • Если имя есть в словаре, то выводит в консоль рост по указанному имени.
  • В противном случае выводит в консоль, что указанное имя не найдено.

Выбран пункт 4:

  • Программа запрашивает с консоли новое имя и рост.
  • Если имя есть в словаре, то выводит в консоль, что такой баскетболист уже присутствует в словаре.
  • В противном случае добавляет имя и рост в словарь.

Выбран пункт 5:

  • Программа запрашивает с консоли имя.
  • Если имя есть в словаре, то удаляет баскетболиста из словаря.
  • В противном случае выводит в консоль, что указанное имя не найдено.

Выбран пункт 6:

  • Программа запрашивает с консоли имя.
  • Если имя есть в словаре, то запрашивает с консоли рост и меняет его у баскетболиста.
  • В противном случае выводит в консоль, что указанное имя не найдено.

Домашнее Задание 38.5

Реализуйте следующее меню: 

1. Показать все книги
2. Найти книгу по идентификатору
3. Добавить новую книгу
4. Удалить книгу по идентификатору
5. Изменить информацию по идентификатору
6. Завершить программу

Данные о книге хранятся в виде словаря с ключами "book_id", "title", "author", "genre", "release_year". Например:

book_1 = {"book_id": "B-001", "title": "Python Tricks. The Book", "author": "Dan Bader", "genre": "Reference work", "release_year": 2017}
book_2 = {"book_id": "B-005", "title": "Blood, Sweat and Pixels", "author": "Jason Schreier", "genre": "Video Games", "release_year": 2017}
book_3 = {"book_id": "B-003", "title": "Structure and Interpretation of Computer Programs", "author": "Harold Abelson and Gerald Jay Sussman", "genre": "Programming", "release_year": 1985}

Все книги хранятся в списке. Например:

book_list = [book_1, book_2, book_3]

Изначально список книг может быть пустым.

Выбран пункт 1:

  • Программа выводит в консоль книги в следующем виде: 
    {идентификатор} {название} {автор} {жанр} {год}
{идентификатор} {название} {автор} {жанр} {год}
...

Выбран пункт 2:

  • Программа запрашивает с консоли идентификатор книги.
  • Если введённый идентификатор совпадает с идентификатором одной из книг из списка, то выводит в консоль данные найденной книги в следующем виде: 
    {идентификатор} {название} {автор} {жанр} {год}

Выбран пункт 3:

  • Программа запрашивает с консоли идентификатор, название книги, имя автора, жанр и год выпуска.
  • Формирует из данных словарь.
  • Если введённый идентификатор не совпадает ни с одним идентификатором книги из списка, то добавляет сформированный словарь в список книг.
  • В зависимости от результата выводит в консоль сообщение.

Выбран пункт 4:

  • Программа запрашивает с консоли идентификатор.
  • Если введённый идентификатор совпадает с идентификатором одной из книг из списка, то удаляет найденную книгу из списка книг.
  • В зависимости от результата выводит в консоль сообщение.

Выбран пункт 5:

  • Программа запрашивает с консоли идентификатор.
  • Если введённый идентификатор совпадает с идентификатором одной из книг из списка, то делает копию найденной книги и выводит в консоль следующее: 
    Старые данные: {идентификатор} {название} {автор} {жанр} {год}
Новые данные: {идентификатор} {название} {автор} {жанр} {год}
1. Изменить идентификатор
2. Изменить название книги
3. Изменить имя автора
4. Изменить жанр
5. Изменить год выпуска
6. Подтвердить изменения и вернуться
7. Отменить изменения и вернуться
  • Выбран пункт 1-5:
    • Программа запрашивает с консоли соответствующие данные.
    • Изменения происходят с копией.
  • Выбран пункт 6:
    • Программа проверяет идентификатор изменённых данных.
    • Если идентификатор копии совпадает с идентификатором старой книги или же идентификатор копии не совпадает ни с одним идентификатором книги из списка, то заменяет у старой книги все данные на соответствующие из копии и возвращается в главное меню.
    • В зависимости от результата выводит в консоль сообщение.

Необязательно:

  1. Разделите программу на фронтенд и бэкенд. Для этого создайте разные функции. Помните, что во фронтенд помещается логика ввода, проверки ввода и его преобразование, а также вывод. В бэкенд помещается вся остальная логика – поиск, добавление, удаление, изменение, формирование данных для вывода и подобные операции.
  2. Переделайте программу так, чтобы книги хранились не в списке, а в словаре, где ключом является идентификатор книги, а значением – книга как словарь. При этом в самой книге уберите ключ "book_id". В итоге должна получиться следующая структура: 
    {"идентификатор": {"title": "название", "author": "имя", "genre": "жанр", "release_year": 2000},
"идентификатор": {"title": "название", "author": "имя", "genre": "жанр", "release_year": 2003},
 ...}
    Вместе с этим переделайте операции вывода книг, поиска, добавления, изменения и удаления книг.
